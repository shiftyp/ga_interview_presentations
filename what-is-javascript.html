What is JavaScript?

  • JavaScript is a programming language.
  • It is executed in the browser in the context of a web page.
  • It's awesome.

JavaScript is like a Written Language

  • It has grammatical rules called syntax.
  • It has constructs like clauses, sentences, and paragraphs called expressions, statements, and blocks.
  • It is read or interpreted one construct at a time.
  • It has verbs, adjectives, and nouns which can be keywords, operators, constants, or variables.
  • It has punctuation and symbols that change the meaning of expressions.
  • It can be used to reference abstract constructs.
  • It has changed over time to better describe it's environment.
  • If you don't know it, it's hard to read it.
  • It can be translated into plain English.

JavaScript is also Unique

  • All expressions are imperatives.
  • A poem in JavaScript is a haiku, not an epic.
  • JavaScript is a language of logic, and can describe it in a very succinct way.
  • JavaScript relies heavily on symbols.
  • In JavaScript, you can write something that is valid, but almost unintelligable.
  • In JavaScript we don't just describe things, we can create things.
  • There are literals; which don't reference a thing, but actually are the thing.
  • Control flow makes a JavaScript program like a choose your own adventure novel.

Expressions and Operators

  • An expression is a bit of code that produces a value.
  • Values come in many of types, but the basic ones are called primitives.
    • number (integer and floating point)
    • string
    • boolean
    • null
    • undefined
  • Operators are special symbols or words that indicate some action.
  • Some of them are used in expressions to determine a value.

Some Simple Expressions with numbers.

  • A number in javascript can be an integer (a whole number) or a float (a number with a decimal point).
  • The basic operators that act on numbers are the plus ( + ), multiplication ( * ), subtraction ( - ) and division ( / ) operators.
  • The following expressions make use of numbers and the plus operator.
  • {% highlight javascript %} // An integer value: 1 1 {% endhighlight %}
  • {% highlight javascript %} // Also an integer value: 2 1 + 1 {% endhighlight %}
  • {% highlight javascript %} // A floating point value: 0.1 0.1 {% endhighlight %}
  • {% highlight javascript %} // Also a floating point value: 1.1 0.1 + 1 {% endhighlight %}

Some Simple Expressions with strings.

  • A string is a value which represents text.
  • Strings are distinguished from code using single or double quotes.
  • We can combine or "concatenate" strings using the plus ( + ) operator.
  • {% highlight javascript %} // A String value: Hello World! 'Hello World!' {% endhighlight %}
  • {% highlight javascript %} // Also a string value: Hello World! 'Hello ' + 'World!' {% endhighlight %}
  • {% highlight javascript %} // Also String value: 1 "1" {% endhighlight %}
  • {% highlight javascript %} // Also a string value: 1 is the loneliest number... 1 + ' is the loneliest number'; {% endhighlight %}

Some Simple Expressions with booleans.

  • A boolean has two possible values, true and false.
  • We use booleans all over the place for determining what to do in different situations.
  • We can combine booleans using the "or" ( || ) and "and" ( && ) operators.
  • We can reverse the value of a boolean by using the not ( ! ) operator.
  • {% highlight javascript %} // A boolean value: true true {% endhighlight %}
  • {% highlight javascript %} // Also a boolean value: true true || false {% endhighlight %}
  • {% highlight javascript %} // Also a boolean value: false true && false {% endhighlight %}
  • {% highlight javascript %} // Also a boolean value: false !true {% endhighlight %}

A Simple Expression with null

  • null is a value that is sort of the opposite of every other value.
  • We often use null to indicate that the value for something doesn't exist.
  • {% highlight javascript %} // A null value (this is all you get for now). null {% endhighlight %}

A Simple Expression with undefined

  • undefined is the absence of a value.
  • It's different from null, since null is actually a type of value.
  • Things that are undefined are often the source of errors.
  • {% highlight javascript %} // undefined (this is all you get for now). undefined {% endhighlight %}

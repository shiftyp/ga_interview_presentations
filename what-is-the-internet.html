What is the internet?

  • The internet is old, maybe older than you think.
  • The internet started out as a Department of Defense project called ARPANET in 1969.
  • What most people think of as the internet is a portion of it called the World Wide Web.
  • The Web came about in the late 1980s and early 1990s.
  • The Web created a graphical interface to the internet, and the website was born.
  • The web and internet still function much the same way today.

Servers and Clients

  • The internet is a complex system, but from our standpoint it has two basic components.
    • Clients which make requests.
    • Servers which respond to requests.

Servers

  • In web development we often refer to servers as the "backend" of an application.
  • Servers are computers or programs that respond to requests from clients, and return a result.
  • This result can contain anything from a website, to a file, to raw data.
  • In the early internet there were just four computers that acted as servers.
  • It's hard to guess how many servers there are today, but in 2012 there were 634 Million websites.

Clients

  • In web development we often refer to clients as the "frontend" of the application.
  • For our purposes, the client can refer to the computer, the browser, or the code that runs on the browser.
  • It's really hard to guess how many clients there are today, but in 2012 there were an estimated 2.5 Billion internet users.

Addressing Servers and Clients

  • Clients and servers interact through protocols.
  • The basic protocols for the internet are called TCP/IP.
  • Servers and clients are refered to using numerical "IP" addresses.
  • We rarely use these for servers, because there are special servers that act as phone books called "Domain Name Servers".
  • These allow us to use human readable "Domain Names".
  • 74.125.228.69 -> google.com

HyperText Transfer Protocol

  • The Web has it's own protocol, called HyperText Transfer Protocol or HTTP.
  • This defines the system of requests and responses that allow web clients and servers to interact.
  • There are several types of requests, but the basic ones are called GET and POST.
    • A GET is used to request websites, files, and data.
    • A POST is used to return some data to the server.

Anatomy of a Request and Response

  • Requests and responses consist of headers and a body.
  • The headers contain information that say how to handle the request / response.
    • host
    • status
    • method
  • The body contains the website, file, or data to be transferred.

Web Browsers

  • Web Browsers are the type of clients we are concerned about in frontend development.
  • They are programs that make requests and visually render the resulting data.
  • In addition to files, there are langauges used by websites to render this data.
    • HyperText Markup Language (HTML).
    • Cascading Style Sheets (CSS).
    • JavaScript.
  • How these langauges are handled depends greatly on the browser.
  • There are standards for these languages, but they aren't always followed.
  • Browsers can also natively parse additional languages used to transfer data and images.
    • JavaScript Object Notation (JSON).
    • Extensible Markup Language (XML).
    • Scalable Vector Graphics (SVG).
